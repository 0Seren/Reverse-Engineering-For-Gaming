Part 1: Introduction to Memory

What is memory?

For our purposes we will mainly think of memory as being split into two different types: Non-Volitle Memory (NVM) and Random Access Memory (RAM).

NVM is what we'll think of as any data that is stored on the hard drive of a computer or on a game disc. This data all makes use of the persistant storage of NVM and includes things such as the program's source code, save files, and character models/textures. Typically much of what is stored in NVM is eventually loaded into RAM to be used and manipulated as needed.

RAM is where a computer stores all data about a program that is currently running. This includes all traditional data, such as a player's location and health value, as well as "non-tradtional" data, which is typically the application's code/instruction set. For now we will be focusing mainly on traditional data in RAM.

Ones and Zeros

All data in a computer is stored with 0's and 1's combined in different ways. We call these 0's and 1's bits. And we call a collection of 8 bits a byte. Even though there are some data types that could, theoretically, be stored as a single bit, the smallest unit that is typically used to store a peice of data is a byte, so this is what we'll be focusing on. Another common piece of terminology is the word. For our purposes, we'll define a word as 32 bits or, more commonly, 4 bytes.

Addresses

So now that we know what types of data are available, let's take a look at how these primitive types might be stored in memory. First off, let's take a look at how these data types are used in a programming language (C++). 

    int health =  100;
    bool is_dead = false;
    float x_position = 13.37;
Here we've declared and initialized three variables all of different types, which we give human readable names of health, is_dead, and x_position. While this is fantastic for us as we can refer to these names anytime we want and work with them that way, this doesn't work so well for the program/application using these variables. Instead the application must refer to each of our variables with a more computer friendly "name", called an address. These addresses are useful to the program because they are assigned so that they always correspond to a location in memory (RAM). For instance, the address for health may be 1A0C84 (the prefix represents that we are using hexadecimal), and when the program goes to that location in memory it will find the value of health. In a sense, a name of a variable is just an alias for its address. The following table shows how this looks:

Value 100 false 13.37
Address 1A0C84 1A0C88 1A0C89
Name health is_dead x_position
Quick Note: Notice how the addresses are spaced apart according to how many bytes each data type takes up (Integer takes 4, Boolean 1).
